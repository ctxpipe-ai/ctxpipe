# Session summaries

Short **episodic** notes so the next session (or another agent) can resume without replaying the whole chat.

## Naming

`YYYY-MM-DD-topic-slug.md`

If multiple sessions the same day, add time: `YYYY-MM-DD-HHMM-topic-slug.md`.

## What to include

- **Summary** (a few sentences: what was done, what matters next)
- **Work completed** (bullets)
- **Decisions** (ADR refs if any)
- **Context for next session** (files, branches, open questions)

Keep each file **compact**; detailed specs belong in ADRs or `patterns.md`, not session logs.

Create a file when ending a substantive session or when using the **session-handoff** skill.
