## ADR 0007 - OpenCypher Graph DB and FalkorDB as Default

- **Status**: Accepted
- **Date**: 2026-03-05

### Context

We need an OpenCypher-compatible graph database for future graph workloads (knowledge graphs, RAG, etc.). The backend is multi-tenant (org-scoped); graph data must be isolated per organization. We must support multiple engines (FalkorDB, Neo4j, Memgraph, Neptune) with a **reliable, unified multi-tenancy strategy** across all of them.

Cypher queries will be generated by an LLM. We cannot trust the LLM to correctly scope queries—tenant isolation must be enforced at the database layer.

### Decision

1. **Support all OpenCypher databases** (Neo4j, FalkorDB, Memgraph, Neptune) via a single Bolt-compatible driver and configurable `GRAPH_DB_URI`. FalkorDB is the **default** for local dev and recommended for production.

2. **Why FalkorDB as default**:
   - **Cost**: Open-source, no Enterprise tier; Neo4j multi-database is Enterprise-only.
   - **DX**: Simple Docker setup, Bolt protocol, fast iteration.
   - **Native multi-tenancy**: Multiple isolated graphs per instance, 10,000+ graphs, strict isolation.

3. **Unified multi-tenancy strategy** — two modes only:

| Engine           | Database-per-tenant | Instance-per-tenant | Notes                              |
| ---------------- | ------------------- | ------------------- | ---------------------------------- |
| FalkorDB         | Yes (native)        | Yes                 | Graphs created on first use        |
| Neo4j Enterprise | Yes                 | Yes                 | `CREATE DATABASE` required (admin) |
| Memgraph         | Yes (Enterprise)    | Yes                 | `CREATE DATABASE` required (admin) |
| Neo4j Community  | No                  | Yes                 | Single DB per instance             |
| Neptune          | No                  | Yes                 | Single graph per cluster           |

**Mode A — Database-per-tenant** (shared instance): One URI, multiple orgs. Pass `database: "org_{orgId}"` to driver. FalkorDB auto-creates graphs; Neo4j Enterprise / Memgraph require `CREATE DATABASE org_xxx` (admin). **Safe for LLM-generated Cypher.**

**Mode B — Instance-per-tenant** (separate instance per org): Each org has its own URI. Connection layer resolves to the driver for that org's instance. Per-org URI via `GRAPH_DB_URI_<orgSlug>` (e.g. `GRAPH_DB_URI_acme`, `GRAPH_DB_URI_piedpiper`). No fallback to `GRAPH_DB_URI`. **Safe for LLM-generated Cypher.**

**Tenancy strategy** is **inferred** from provider (no env var). FalkorDB/Neo4j Enterprise/Memgraph → `database`; Neo4j Community/Neptune → `instance-per-tenant`.

### Consequences

- Single API (`executeInOrgContext`) works across FalkorDB, Neo4j, Memgraph, and Neptune.
- Both strategies are safe for LLM-generated Cypher.
- Existing ADRs (0001, 0003) remain unchanged.
