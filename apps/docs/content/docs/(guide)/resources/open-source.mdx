## Repository

Source code lives at **[github.com/ctxpipe-ai/ctxpipe](https://github.com/ctxpipe-ai/ctxpipe)** —
a pnpm + Turborepo monorepo with:

- **`apps/backend`** — Hono (Bun), REST + MCP, Better Auth, Drizzle, LangGraph workflows
- **`apps/ui`** — TanStack Start (React + Vite)
- **`apps/codesearch`** — Zoekt orchestration and repo/file API
- **`apps/docs`** — this documentation site (Next.js + Fumadocs)
- **`apps/otel-collector`** — reference OpenTelemetry Collector config

## Hosted vs OSS

The **hosted** product at [ctxpipe.ai](https://ctxpipe.ai) runs the same architectural
ideas as the open-source tree: you can **self-host** for full control of data and
infra, or use hosted for operational convenience.

## Contributing

Issues and pull requests are welcome on GitHub. For large changes, skim the ADRs in
`.ai/memory/decisions/` so proposals align with recorded architecture decisions.

## Security

Report sensitive issues through the contact channels published on
[ctxpipe.ai](https://ctxpipe.ai) or GitHub security advisories if enabled for the
organization — do not post secrets in public issues.
