This page is about **product integrations** — wiring ctx| to the SaaS and
systems your team already uses (issue trackers, docs, incidents, observability).
It is **not** about how **AI clients** talk to ctx|; for that, see
[ctx| MCP](/docs/mcp/mcp-docs) (Streamable HTTP, API keys, and OAuth for agents).

## Available today

| Area | What you can connect | Where to go |
| --- | --- | --- |
| **Code & repos** | **GitHub** via the GitHub App — choose orgs/repos, ingestion, and PR-based workflows tied to your ctx\| organization. | [Connection overview](/docs/connections) and [Ingestion](/docs/connections/ingestion) |
| **Documentation** | **Confluence** via the Atlassian / Forge connector (spaces and pages). The **Connectors** page lists every **connection** (`con_*` id); you can have multiple GitHub or Forge installs per org. Sync targets reference a **repository id** from Repositories. | [Confluence connector](/docs/connections/confluence-connector) |

If something is not listed here, it may be on our roadmap. [Get in touch with us](/docs/getting-started/get-in-touch) to find out.

## On the roadmap

We intend to add connectors for common engineering tools so context stays
**organization-scoped** and **explicit** (you choose what is linked), rather
than ad-hoc scraping. Examples under active or planned consideration include:

- **Issue & project tracking** — e.g. Linear, Jira
- **Incidents & on-call** — e.g. PagerDuty
- **Docs & wikis** — e.g. Notion (beyond repo-native docs)
- **Error tracking & observability** — e.g. Sentry

Roadmap and availability will vary by **hosted vs self-hosted** deployment. The
hosted app provides managed OAuth and install links; self-hosted deployments
often need operator-owned credentials first. For Confluence, see
[Self-host Confluence & Atlassian](/docs/self-hosting/confluence-atlassian).

## For agent platforms (MCP)

Cursor, CI, and other **MCP clients** do not use the same “Connect” flows as
Linear or GitHub in the UI. They call ctx| at `/mcp?orgSlug=...` with **API keys**
or **OAuth** as documented on the [MCP](/docs/mcp/mcp-docs) page.

## REST (optional)

Org-scoped REST under `/:orgSlug/api/v1/...` complements connectors for
automation. Interactive reference: `/.docs/api-reference` on your deployment.
