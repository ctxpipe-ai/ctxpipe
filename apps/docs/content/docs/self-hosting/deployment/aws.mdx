AWS deployments should keep the same product shape as the reference stack while
using AWS services for the runtime, database, secrets, logging, and ingress.

## Reference shape

| ctx\| component | AWS mapping |
|---|---|
| Public backend origin | Application Load Balancer, API Gateway, or another HTTPS ingress in front of `backend` |
| `backend` | ECS service, EKS deployment, App Runner service, or EC2 service |
| `ui` | Private service behind the backend proxy, or a separately deployed service with the same public API origin |
| `worker` | ECS service, EKS deployment, or EC2 worker process |
| `codesearch` | Container service with persistent storage for clone cache and Zoekt index |
| Postgres | Amazon RDS for PostgreSQL 17, or another managed Postgres 17-compatible service |
| Graph database | Self-run FalkorDB, Neo4j, or Memgraph, or Amazon Neptune if your tenancy model matches its single-graph behavior |
| Secrets | AWS Secrets Manager or SSM Parameter Store |
| Logs and traces | CloudWatch Logs plus OTLP export if you use a collector |

## Ingress and URLs

ctx| expects one public backend origin for browser, REST API, auth, connector
callbacks, and MCP:

```bash
AUTH_BASE_URL=https://ctx.example.com
CTXPIPE_PUBLIC_APP_URL=https://ctx.example.com
```

Route the public origin to the backend service. Keep UI, worker, codesearch,
Postgres, and graph database endpoints private unless your architecture has a
specific reason to expose them.

## Stateful services

Plan these before inviting users:

- RDS backups, restore testing, maintenance windows, and storage alarms.
- Persistent storage for `codesearch` repository cache and Zoekt indexes.
- Backup or snapshot procedures for the selected graph database.
- Secret rotation for auth, connector credentials, model keys, and SMTP.
- Observability for backend, worker, and codesearch failures.

## Connectors

GitHub and Confluence both depend on public callback URLs. Make sure the public
origin is reachable from GitHub and Atlassian before testing connector setup.

| Connector | AWS concern |
|---|---|
| GitHub | Webhook URL reaches `{AUTH_BASE_URL}/api/v1/webhook/github/<connectionId>` |
| Confluence | Atlassian OAuth callback reaches `{AUTH_BASE_URL}/api/v1/integrations/atlassian/callback` |
| MCP | Agents can reach `{AUTH_BASE_URL}/mcp?orgSlug=<org-slug>` |
