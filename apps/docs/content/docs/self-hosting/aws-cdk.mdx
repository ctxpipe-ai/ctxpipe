`@ctxpipe-ai/aws-cdk` provides a single high-level construct, `CtxPipe`, for AWS self-hosting.

## What gets provisioned

- ECS Fargate services for `backend`, `worker`, `ui`, and `codesearch`
- Public ALB routing to backend only
- Aurora PostgreSQL for `DATABASE_URL`
- Neptune for graph workloads (`GRAPH_DB_PROVIDER=neptune`)
- EFS mounted at `/data` for codesearch persistence
- SES SMTP credentials + sender identity wiring for transactional email
- Secrets Manager wiring for runtime secrets

## Required configuration

`CtxPipe` requires three prop groups:

- `customDomain` (Route 53 hosted zone ID + ACM certificate + FQDN)
- `modelProvider` (OpenAI-compatible base URL, API key, default model)
- `orgSlug` (single-tenant org slug for `GRAPH_DB_URI_<orgSlug>`)

`AUTH_SECRET` is generated automatically in Secrets Manager by the construct.

The stack intentionally does not boot without model provider credentials.

## Optional configuration

- `connectorSecrets`: GitHub/Atlassian deployment-wide secrets (second deploy pattern)
- `serviceImageTag`: optional override for GHCR image tags (defaults to the commit SHA baked into the package at build time)

Transactional email sender is fixed to `ctxpipe-noreply@<hosted-zone-apex>`.

By default, services pull public GHCR images from `ghcr.io/ctxpipe-ai/*`.

## Install

GitHub Packages serves npm tarballs only to authenticated clients. A **public** package still requires a token with **`read:packages`** (anonymous installs get `401 Unauthorized`).

1. Configure npm for the `@ctxpipe-ai` scope:

```bash
echo "@ctxpipe-ai:registry=https://npm.pkg.github.com=${GH_TOKEN}" >> ~/.npmrc
```

2. Authenticate without committing secrets:
   In GitHub Actions, set the **`GH_TOKEN`** secret (or map `secrets.GITHUB_TOKEN` into `GH_TOKEN` when `permissions: packages: read` applies).

3. Install the package:

```bash
pnpm add @ctxpipe-ai/aws-cdk aws-cdk-lib constructs
```
